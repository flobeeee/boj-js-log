# 수 정렬하기 / 2750

> 📎 링크: https://www.acmicpc.net/problem/2750
> 🏷 유형: Sort
> ✅ 결과: 풀었음

---

## 풀이 아이디어

정렬하고 숫자 사이사이에 엔터 넣기 

## 코드

```js
const input = require('fs').readFileSync('/dev/stdin', 'utf8').trim().split('\n');

const n = Number(input[0]);
const nums = input.slice(1, n + 1).map(Number);

console.log(nums.sort((a,b) => a - b ).join('\n'))
```

## 시간복잡도

- 내 풀이: O(n log n)
- 이유: sort 가 사용하는 TimSort 알고리즘의 시간복잡도

## 막혔던 부분

- 백준에서 인풋값을 가져올 때 기본적으로 쓰는 소스가 뭔지 모름
- const input = require('fs').readFileSync('/dev/stdin', 'utf8').trim().split('\n'); 사용함

- `sort()` 기본 동작이 문자열 기준이라 숫자 정렬 시 틀린 결과가 나옴
- `(a, b) => a - b` 비교 함수를 넘겨야 숫자 오름차순 정렬됨

## 다른 풀이 비교
